# Kinematics Motion Generator

## Basics Framework

### **Training**
1. **Label existing retargeted kinematics data** with the velocity command using some heuristic.  
2. **Train a model** to generate future key body pose/twist reference trajectories conditioned on the current state and command.  
3. **Generate key body pose/twist reference trajectories** given random commands and save them in a dataset.  
4. **Train a low-level motion tracking policy** to track the entire dataset.  

### **Deployment**
- **User input:** Joystick command (velocity).  
- **High-level Kinematics Motion Generator:** Generates future key body pose/twist trajectories in real-time (1–5 Hz).  
- **Low-level Motion Tracking Policy:** Tracks the generated key body pose/twist trajectories (50 Hz).  

---

## One-Step Further

1. **Close the loop:** Add simulation rollouts to augment the mocap dataset.  
2. **Condition on obstacle:** Incorporate environment information (e.g., obstacle positions) when generating motion.  
3. **Different user input interface:** Use VR devices to command end-effector poses.
